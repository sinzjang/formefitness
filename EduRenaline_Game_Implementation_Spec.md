# EduRenaline — 인지 훈련 게임 구현 명세서 (AI 개발용)

> **목적**: 이 문서는 AI 코딩 에이전트(Claude Code, Cursor 등)에게 게임 구현을 지시하기 위한 명세서입니다.
> **출처**: Elevate(44개) + BrainHQ(29개) 게임 분석 기반. 특허 우회 구현 방식 적용.
> **원칙**: 모든 게임은 공개 인지과학 패러다임(n-back, Go/No-Go, MOT, Stroop, Mental Rotation 등)을 기반으로 하며, 자극 소재·UI·난이도 로직은 EduRenaline 자체 설계로 특허/저작권을 회피합니다.

---

## 0. 이 문서 사용법 (AI 에이전트용 지침)

각 게임 명세는 다음 구조를 따릅니다:

```
### 게임명
- 우선순위 점수 / 인지 능력 / 패러다임 출처
- 한 줄 컨셉
- 게임 플레이 흐름 (단계별)
- 구현 명세 (입력/출력/상태/난이도 로직)
- 데이터 구조 (JSON 스키마)
- 특허 우회 포인트
```

**AI 에이전트는 다음을 준수**:
1. 우선순위 P0 게임부터 구현. P0 → P1 → P2 순서.
2. 모든 게임은 공통 엔진(난이도 조절, 점수, 세션 관리)을 공유하도록 설계.
3. 자극 소재는 원본 앱과 다르게 — 일러스트/도형/이모지/TTS 사용.
4. 난이도 조절은 단순 정답률 기반(공개 도메인): 연속 정답 3개 → 레벨업, 연속 오답 2개 → 레벨다운.

## 1. 우선순위 시스템

### 1.1 점수 산정 기준

| 기준 | 가중치 | 설명 |
|------|--------|------|
| **접근성 (Accessibility)** | ×3 | 룰 이해 난이도. 5초 안에 이해 가능하면 높음. 시니어·저연령 접근성 |
| **효과 (Effect)** | ×3 | 전이 근거. far transfer 확인 or 실용 스킬 직접 향상 = 높음 |
| **구현 용이성 (Build Ease)** | ×2 | MVP 개발 난이도. 낮을수록 우선 |
| **특허 안전성 (Patent Safety)** | ×2 | 공개 패러다임 기반일수록 높음 |

각 항목 1~3점. **총점 = (접근성×3) + (효과×3) + (구현용이성×2) + (특허안전성×2)**. 최대 30점.

### 1.2 우선순위 등급

| 등급 | 점수 | 의미 | 개발 시점 |
|------|------|------|----------|
| **P0** | 26~30 | 즉시 구현. 접근성·효과·구현 모두 우수 | MVP 1차 |
| **P1** | 21~25 | 2차 확장. 가치 높으나 구현 or 접근성 약간 높음 | MVP 2차 |
| **P2** | 16~20 | 선택적. 차별화·다양성 목적 | 추후 |
| **P3** | ~15 | 보류. 구현 난이도 높거나 효과 불확실 | 미정 |

### 1.3 카테고리별 우선순위 (전체 요약)

> **EduRenaline 코어 정체성**: 주의력 · 작업기억 · 집행기능 · 오디오 퍼스트
> 아래 카테고리 순서는 **"쉬운 접근성 + 즉각 효과 체감"** 우선순위입니다.

| 순위 | 카테고리 | 이유 | 대표 P0 게임 |
|------|----------|------|-------------|
| **1** | **처리 속도 (Processing Speed)** | 진입 가장 쉬움. 효과 즉각 체감. far transfer 근거 강함 (운전 등) | 이중 주의 확장, 빠른 판별 |
| **2** | **수 연산 (Math)** | 룰 직관적. 전 연령. 실용 스킬 직접 향상 | 빠른 연산, 추정 |
| **3** | **작업기억 (Working Memory)** | EduRenaline 코어. 오디오 활용 강점 | 순서 기억, 청각 n-back |
| **4** | **주의력/억제 (Attention/Inhibition)** | 집중력 체감 좋음. Go/No-Go 단순 | 정지 반응, 선택적 주의 |
| **5** | **언어/어휘 (Language)** | 실용 스킬. 단 영어권 의존도 높음 (한국어화 필요) | 문맥 추론, 어휘 매칭 |
| **6** | **공간/내비게이션 (Spatial)** | 차별화 요소. 구현 다소 복잡 | 정신적 회전 |
| **7** | **사회/감정 (Social)** | 니치. 우선순위 낮음 | 감정 인식 |

## 2. P0 게임 — 즉시 구현 (MVP 1차)

> 접근성·효과·구현·특허 모두 우수. 이 게임들로 MVP 출시 권장.

---

### 2.1 ⚡ Rapid Match (빠른 판별) — 처리 속도

- **우선순위**: P0 (29/30) | **인지 능력**: 처리 속도, 시각 변별 | **패러다임**: Speed of Processing (공개)
- **원본 참고**: Elevate Agility / BrainHQ Eye for Detail
- **접근성** ⭐⭐⭐ | **효과** ⭐⭐⭐ | **구현** ⭐⭐⭐ | **특허** ⭐⭐⭐

**한 줄 컨셉**: 화면에 짧게 나타나는 두 도형/아이콘이 같은지 다른지 빠르게 판별.

**게임 플레이 흐름**:
1. 화면 중앙에 기준 자극(도형) 표시 (예: 파란 삼각형)
2. 짧은 시간(시작 800ms) 후 두 번째 자극 제시
3. 사용자: "같음" / "다름" 버튼 탭 (또는 좌/우 스와이프)
4. 정답 시 다음 라운드, 제시 시간 단축
5. 60초 세션, 정답 수 + 평균 반응속도 점수화

**구현 명세**:
```
입력: 탭(같음/다름) 또는 스와이프
출력: 정답 여부, 반응속도(ms), 누적 점수
상태: { currentStimulus, comparisonStimulus, presentationTime, streak, score, sessionTimeLeft }
난이도 로직:
  - presentationTime: 시작 800ms, 연속 정답 3개마다 -50ms (최소 150ms)
  - 자극 유사도: 레벨업 시 색/모양 차이 미세화
  - 방해 요소: 상위 레벨에서 배경 노이즈 추가
점수: (정답 수 × 10) - (오답 수 × 5) + 속도 보너스
```

**데이터 구조**:
```json
{
  "gameId": "rapid_match",
  "stimulusSet": [
    { "id": "s1", "shape": "triangle", "color": "#3B82F6", "size": "md" }
  ],
  "levels": [
    { "level": 1, "presentationMs": 800, "similarity": 0.3, "noise": false }
  ]
}
```

**특허 우회 포인트**: BrainHQ Eye for Detail의 이미지 매칭을 단순 도형 변별로 대체. 자극은 자체 제작 도형. visual discrimination 패러다임은 공개 도메인.

### 2.2 ⚡ Dual Focus (이중 주의 확장) — 처리 속도 ★핵심

- **우선순위**: P0 (28/30) | **인지 능력**: 처리 속도, 유용 시야(UFOV), 분할 주의 | **패러다임**: UFOV (공개)
- **원본 참고**: BrainHQ Double Decision (★far transfer 확인된 유일 게임)
- **접근성** ⭐⭐ | **효과** ⭐⭐⭐ | **구현** ⭐⭐ | **특허** ⭐⭐⭐ (구조 변경 적용)

**한 줄 컨셉**: 중앙 과제를 처리하면서 동시에 주변 시야에 나타난 타겟 위치를 기억.

**게임 플레이 흐름**:
1. 화면 중앙에 도형 한 개 짧게 표시 (예: 빨강/파랑 원 중 무엇?)
2. **동시에** 화면 주변부(8방향 중 1곳)에 타겟 아이콘(별) 깜빡임
3. 자극 사라진 후: ① 중앙 도형 색 선택 ② 별이 있던 방향 탭
4. 둘 다 맞아야 정답. 제시 시간 단축, 별 위치 점점 외곽으로
5. 핵심: **중앙 집중 중에도 주변 정보를 포착**하는 훈련

**구현 명세**:
```
입력: 2단계 — 중앙 선택 → 주변 위치 탭
출력: 중앙 정답, 주변 정답, 동시 성공 여부
상태: { centerStimulus, peripheralPosition(0-7), presentationMs, eccentricity, level }
난이도 로직:
  - presentationMs: 시작 500ms → 최소 100ms
  - eccentricity(중심-주변 거리): 레벨업마다 외곽으로
  - 주변 방해물: 상위 레벨에서 별 외 distractor 추가
점수: 동시 성공만 카운트 (× 15), UFOV 확장 지표 별도 기록
```

**데이터 구조**:
```json
{
  "gameId": "dual_focus",
  "centerStimuli": [{ "id": "c1", "type": "color", "value": "red" }],
  "peripheralPositions": 8,
  "levels": [
    { "level": 1, "presentationMs": 500, "eccentricity": 0.4, "distractors": 0 }
  ]
}
```

**특허 우회 포인트** ⚠️ 중요: Double Decision은 자동차/Route66 비주얼 + 특정 타이밍이 특허. **이중 과제 구조(중앙+주변 동시)는 UFOV 공개 연구 기반**이므로 유지. 자극을 도형/별로 완전 교체. EduRenaline 브랜드 비주얼 적용. ※far transfer 근거가 가장 강한 게임이므로 효과 마케팅 핵심 자산.

### 2.3 ➕ Quick Calc (빠른 연산) — 수 연산

- **우선순위**: P0 (30/30) | **인지 능력**: 처리 속도, 암산, 작업기억 | **패러다임**: Arithmetic Fluency (공개)
- **원본 참고**: Elevate Subtraction/Division/Averages
- **접근성** ⭐⭐⭐ | **효과** ⭐⭐⭐ | **구현** ⭐⭐⭐ | **특허** ⭐⭐⭐

**한 줄 컨셉**: 시간 제한 안에 사칙연산 문제를 빠르게 푼다.

**게임 플레이 흐름**:
1. 연산 문제 제시 (예: 47 - 18 = ?)
2. 숫자패드 또는 4지선다로 답 입력
3. 정답 시 즉시 다음 문제, 콤보 누적
4. 60초 세션. 연산 종류 선택 가능(+, -, ×, ÷, 평균, 퍼센트)
5. 레벨업 시 자릿수 증가, 시간 단축

**구현 명세**:
```
입력: 숫자패드 입력 or 4지선다 탭
출력: 정답 여부, 콤보, 누적 점수
상태: { problem, answer, operationType, level, combo, sessionTimeLeft }
난이도 로직:
  - 자릿수: 레벨1 한자리 → 레벨5 세자리
  - 연산 복잡도: 단일 → 복합(47-18+5)
  - 시간 압박: 문제당 제한시간 점진 단축
점수: (정답 × 10) × 콤보배수, 오답 시 콤보 리셋
모드: ['add', 'subtract', 'multiply', 'divide', 'average', 'percent']
```

**데이터 구조**:
```json
{
  "gameId": "quick_calc",
  "operationModes": ["add", "subtract", "multiply", "divide", "average", "percent"],
  "levels": [
    { "level": 1, "digits": 1, "compound": false, "timePerProblem": 8 }
  ]
}
```

**특허 우회 포인트**: 사칙연산 게임은 완전 공개 도메인. 문제 생성 로직만 자체 구현. 특허 위험 0.

### 2.4 ➕ Number Sense (추정) — 수 연산

- **우선순위**: P0 (27/30) | **인지 능력**: 근사 수 체계(ANS), 암산 추정 | **패러다임**: ANS (공개)
- **원본 참고**: Elevate Estimation
- **접근성** ⭐⭐⭐ | **효과** ⭐⭐⭐ | **구현** ⭐⭐⭐ | **특허** ⭐⭐⭐

**한 줄 컨셉**: 여러 숫자의 합/값을 정확히 계산하지 않고 빠르게 추정해 슬라이더로 제출.

**게임 플레이 흐름**:
1. 숫자 묶음 제시 (예: 23 + 48 + 31)
2. 0~200 슬라이더(숫자선) 표시
3. 정확한 답 대신 **빠른 추정값**을 슬라이더로 드래그
4. 정답 ±오차범위 내면 성공. 범위는 레벨업 시 좁아짐
5. 정밀 계산이 아닌 "직관적 수 감각" 훈련

**구현 명세**:
```
입력: 슬라이더 드래그 → 값 제출
출력: 제출값, 정답, 오차, 성공 여부
상태: { numbers[], correctAnswer, sliderRange, tolerance, level }
난이도 로직:
  - tolerance(허용 오차): 레벨1 ±15% → 레벨5 ±5%
  - 숫자 개수: 2개 → 5개
  - 슬라이더 범위 확대
점수: (1 - 오차율) × 20, 시간 보너스
```

**특허 우회 포인트**: ANS 추정 + 슬라이더 UI는 공개 패러다임. 특허 위험 없음.

### 2.5 🧠 Sequence Memory (순서 기억) — 작업기억 ★오디오

- **우선순위**: P0 (27/30) | **인지 능력**: 작업기억, 순서 기억, 청크 | **패러다임**: Simon / Corsi (공개)
- **원본 참고**: BrainHQ Rhythm Recall/Syllable Stacks / Elevate Sequencing
- **접근성** ⭐⭐⭐ | **효과** ⭐⭐ | **구현** ⭐⭐⭐ | **특허** ⭐⭐⭐

**한 줄 컨셉**: 제시된 순서(소리+빛)를 기억해 똑같이 따라 입력. 점점 길어진다.

**게임 플레이 흐름**:
1. 패드 4~6개 화면 배치
2. 시스템이 순서대로 패드 점등 + **고유 사운드 재생** (오디오 퍼스트)
3. 사용자가 동일 순서로 패드 탭
4. 성공 시 시퀀스 +1 길이 증가
5. 청각·시각 동시 자극으로 기억 강화

**구현 명세**:
```
입력: 패드 탭 시퀀스
출력: 시퀀스 일치 여부, 최대 도달 길이
상태: { pads[], sequence[], userInput[], currentLength, mode }
난이도 로직:
  - 시퀀스 길이: 시작 2 → 무제한 (실패 시 종료)
  - 제시 속도: 레벨업 시 빨라짐
  - 모드: 'audio_visual'(기본) / 'audio_only'(고난도) / 'visual_only'
점수: 최대 도달 시퀀스 길이 × 15
```

**데이터 구조**:
```json
{
  "gameId": "sequence_memory",
  "padCount": 6,
  "sounds": ["tone_c", "tone_d", "tone_e", "tone_f", "tone_g", "tone_a"],
  "modes": ["audio_visual", "audio_only", "visual_only"],
  "startLength": 2
}
```

**특허 우회 포인트**: Simon 게임 방식은 완전 공개. 오디오 퍼스트 강화는 EduRenaline 차별점. 특허 위험 없음.

### 2.6 🧠 Audio N-Back (청각 n-back) — 작업기억 ★오디오 코어

- **우선순위**: P0 (26/30) | **인지 능력**: 작업기억, 유동 지능 | **패러다임**: n-back (공개, Jaeggi 2008)
- **원본 참고**: BrainHQ Auditory Ace
- **접근성** ⭐⭐ | **효과** ⭐⭐ (near transfer 확실) | **구현** ⭐⭐⭐ | **특허** ⭐⭐⭐

**한 줄 컨셉**: 들려주는 자극이 N개 전 자극과 같은지 판별. 작업기억의 대표 훈련.

**게임 플레이 흐름**:
1. 일정 간격으로 오디오 자극 재생 (글자/숫자/소리 TTS)
2. 현재 자극이 **N단계 전** 자극과 같으면 "일치" 버튼 탭
3. 1-back으로 시작, 정확도 높으면 2-back, 3-back으로
4. EduRenaline 오디오 퍼스트 정체성에 완벽 부합

**구현 명세**:
```
입력: "일치" 버튼 탭 (일치할 때만)
출력: hit/miss/false-alarm, 정확도
상태: { stimulusStream[], nLevel, currentIndex, responses[] }
난이도 로직:
  - nLevel: 1 → 2 → 3 (정확도 80% 이상 시 상승, 50% 이하 시 하락)
  - 자극 간격: 레벨업 시 짧아짐
  - 자극 종류: 단순(글자) → 복합(글자+위치 dual n-back)
점수: (hits × 10) - (false_alarms × 5), n레벨 보너스
```

**데이터 구조**:
```json
{
  "gameId": "audio_nback",
  "stimulusType": "letter",
  "stimulusPool": ["가", "나", "다", "라", "마", "바"],
  "startN": 1,
  "interStimulusInterval": 2500
}
```

**특허 우회 포인트**: n-back은 학술 공개 패러다임. BrainHQ의 플레이카드 비주얼 대신 한국어 음절 TTS 사용. 특허 위험 없음.

---

### 2.7 🎯 Stop Signal (정지 반응) — 주의력/억제

- **우선순위**: P0 (26/30) | **인지 능력**: 반응 억제, 집행기능, 지속 주의 | **패러다임**: Go/No-Go (공개)
- **원본 참고**: BrainHQ Freeze Frame
- **접근성** ⭐⭐⭐ | **효과** ⭐⭐ | **구현** ⭐⭐⭐ | **특허** ⭐⭐⭐

**한 줄 컨셉**: 대부분의 자극엔 빠르게 반응하되, 특정 "정지 신호"엔 반응을 참는다.

**게임 플레이 흐름**:
1. 화면에 자극(예: 초록 원) 연속 등장 → 빠르게 탭 (Go)
2. 가끔 "정지 신호"(빨강 원) 등장 → **탭하지 않고 참기** (No-Go)
3. Go에 빠르게, No-Go에 정확히 참으면 점수
4. 정지 신호 빈도·유사도 점진 증가로 억제 통제 훈련

**구현 명세**:
```
입력: 탭 (Go에만, No-Go엔 무반응)
출력: go정답률, nogo억제율, 반응속도
상태: { stimulusType(go/nogo), reactionTime, streak, nogoFrequency }
난이도 로직:
  - nogoFrequency: 시작 20% → 점진 변동(예측 어렵게)
  - go-nogo 자극 유사도: 레벨업 시 미세화
  - 자극 제시 속도 증가
점수: (go정답 × 5) + (nogo억제 × 10) - (충동 탭 × 8)
```

**특허 우회 포인트**: Go/No-Go는 신경과학 기본 패러다임(완전 공개). 자극을 도형/색으로 자체 제작. 특허 위험 없음.

---

### 2.8 🎯 Color Clash (선택적 주의) — 주의력/억제

- **우선순위**: P0 (26/30) | **인지 능력**: 선택적 주의, 간섭 억제, 인지 유연성 | **패러다임**: Stroop (공개)
- **원본 참고**: BrainHQ Divided Attention / (Stroop 고전)
- **접근성** ⭐⭐⭐ | **효과** ⭐⭐ | **구현** ⭐⭐⭐ | **특허** ⭐⭐⭐

**한 줄 컨셉**: 단어의 '의미'가 아닌 '글자 색'에 반응 — 간섭을 억제하는 고전 Stroop.

**게임 플레이 흐름**:
1. 색 이름 단어가 다른 색으로 표시 (예: "빨강"이 파란색으로)
2. 사용자는 **글자 색**(파랑)을 선택 — 단어 뜻(빨강) 무시
3. 일치/불일치 조건 섞어 제시
4. 인지적 간섭을 억제하는 집행기능 훈련

**구현 명세**:
```
입력: 색상 버튼 탭 (글자 색 기준)
출력: 정답률, 간섭 조건별 반응속도 차이(Stroop effect)
상태: { word, fontColor, congruent(bool), options[] }
난이도 로직:
  - 불일치 비율 증가
  - 색상 선택지 수: 3 → 5
  - 시간 압박 추가
점수: (정답 × 8) + 속도 보너스, 불일치 정답 가중치 ↑
```

**특허 우회 포인트**: Stroop는 1935년 고전 패러다임(퍼블릭 도메인). 한국어 색 이름 사용. 특허 위험 없음.

## 3. P1 게임 — 2차 확장 (MVP 2차)

> 가치 높으나 구현 복잡도 or 접근성이 P0보다 약간 높음.

---

### 3.1 📖 Context Fill (문맥 추론) — 언어

- **우선순위**: P1 (24/30) | **인지 능력**: 문맥 추론, 어휘 | **패러다임**: Cloze Test (공개)
- **원본 참고**: Elevate Context
- **한 줄 컨셉**: 빈칸 문장에 가장 알맞은 단어를 고른다 (한국어 적용 필수).

**구현 명세**:
```
입력: 4지선다 단어 선택
상태: { sentence, blankIndex, options[], correctIndex }
난이도: 문장 복잡도 ↑, 선택지 유사도 ↑
데이터: 한국어 문장 DB 자체 제작 (공개 코퍼스 기반)
```
**특허 우회**: Cloze test는 완전 범용. 한국어 콘텐츠만 자체 제작.

---

### 3.2 📖 Word Match (어휘 매칭) — 언어

- **우선순위**: P1 (23/30) | **인지 능력**: 어휘 처리 속도, 유의어/반의어 | **패러다임**: Lexical Matching (공개)
- **원본 참고**: Elevate Agility/Synonyms/Inversion
- **한 줄 컨셉**: 단어와 유의어(또는 반의어)를 시간 안에 빠르게 매칭.

**구현 명세**:
```
입력: 카드 매칭 (드래그 or 탭)
상태: { wordCards[], matchType(synonym/antonym), timeLeft }
난이도: 단어 난이도 ↑, 카드 수 ↑, 시간 단축
데이터: 한국어 유의어/반의어 DB (공개 사전 기반)
```
**특허 우회**: 매칭 게임 범용. 한국어 어휘 DB 자체 구축.

---

### 3.3 🧠 Spatial Recall (격자 위치 기억) — 작업기억/공간

- **우선순위**: P1 (23/30) | **인지 능력**: 시공간 작업기억 | **패러다임**: Corsi Block (공개)
- **원본 참고**: BrainHQ Memory Grid/Mental Map
- **한 줄 컨셉**: 격자에서 점등된 칸 순서를 기억해 재현. 변형(회전) 추가 가능.

**구현 명세**:
```
입력: 격자 칸 탭 시퀀스
상태: { gridSize, litSequence[], transform(none/rotate/flip), userInput[] }
난이도: 격자 크기 ↑, 시퀀스 길이 ↑, 변환 추가(고난도)
```
**특허 우회**: Corsi block은 1972년 공개 패러다임. 변환 로직 자체 구현.

---

### 3.4 🎯 Multi-Track (다중 객체 추적) — 주의력

- **우선순위**: P1 (22/30) | **인지 능력**: 분할 주의, 동적 추적 | **패러다임**: MOT (공개, Pylyshyn 1988)
- **원본 참고**: BrainHQ Target Tracker/Scene Crasher
- **한 줄 컨셉**: 여러 움직이는 객체 중 표시된 타겟만 끝까지 눈으로 추적.

**구현 명세**:
```
입력: 정지 후 타겟 객체 탭
상태: { objects[], targetIds[], motionPaths[], objectCount }
난이도: 객체 수 ↑, 속도 ↑, 타겟/방해물 유사도 ↑
```
**특허 우회**: MOT는 공개 패러다임. 객체를 이모지/도형으로. 이동 알고리즘 자체 구현.

---

### 3.5 🔊 Pitch Discriminate (음높이 변별) — 처리 속도 ★오디오

- **우선순위**: P1 (22/30) | **인지 능력**: 청각 처리 속도, 주파수 변별 | **패러다임**: 심리음향학 (공개)
- **원본 참고**: BrainHQ Sound Sweeps/Fine Tuning
- **한 줄 컨셉**: 두 음 중 더 높은 음을 빠르게 판별. 점점 비슷해진다.

**구현 명세**:
```
입력: "첫음/둘째음 높음" 탭
상태: { tone1Freq, tone2Freq, freqDifference, ISI }
난이도: 주파수 차이 ↓, 음 간격(ISI) ↓
구현: Web Audio API로 순음(pure tone) 생성
```
**특허 우회**: 주파수 변별은 공개 심리음향학. Web Audio API 사용. 특허 위험 없음.

| 게임명 | 카테고리 | 패러다임 | 핵심 메커니즘 | 특허 우회 |

| **Mental Rotation** (정신적 회전) | 공간 | Shepard-Metzler (공개) | 두 도형이 회전된 같은 것인지/거울상인지 판별 | 2D 추상 도형 사용 |

## 6. 공통 엔진 명세 (모든 게임 공유)

> AI 에이전트는 게임별 구현 전에 이 공통 엔진을 먼저 구축.

### 6.1 난이도 조절 엔진 (Adaptive Difficulty)
```
공개 도메인 단순 적응 로직 (특허 안전):
  - 연속 정답 3개 → 레벨 +1
  - 연속 오답 2개 → 레벨 -1
  - 레벨 범위: 1 ~ maxLevel (게임별 정의)
※ 복잡한 Bayesian/IRT 적응은 오픈소스 라이브러리 또는 자체 설계
```

### 6.2 세션 & 점수 시스템
```
- 세션: 기본 60초 또는 게임별 정의
- 점수: 게임별 공식, 0~1000 정규화
- 기록: { gameId, date, score, maxLevel, accuracy, avgReactionTime }
- 일일 트레이닝: 추천 게임 3~5개 로테이션
```

### 6.3 인지 능력 매핑 (대시보드용)
```
모든 게임을 5대 능력으로 태깅:
  - PROCESSING_SPEED (처리 속도)
  - WORKING_MEMORY (작업기억)
  - ATTENTION (주의력)
  - EXECUTIVE_FUNCTION (집행기능)
  - LANGUAGE (언어)
→ 사용자 능력별 성장 그래프 제공
```

### 6.4 오디오 퍼스트 원칙 (EduRenaline 차별점)
```
- 모든 게임에 사운드 피드백 (정답/오답/레벨업)
- 작업기억·처리속도 게임은 오디오 모드 우선 제공
- TTS는 한국어 자연스러운 음성 사용
- Audio N-Back, Sequence Memory, Pitch Discriminate가 오디오 코어 게임
```

## 8. 특허/저작권 안전 체크리스트 (구현 시 필수 확인)
